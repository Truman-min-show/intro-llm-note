## 9.1 RAG基础

### 思考：

1. 还记得大一下学期，GPT最开始火爆全网时我对LLM的理解就是一个加强版的搜索引擎，那时我才刚刚学会CPP编程对人工智能和深度学习的理解尚浅。现在有了RAG的实践经验后再去读书突然想到了两年多前的这个想法：LLM结合RAG确实就是更强的搜索引擎，我有看到广大网友也都把大模型当作新的搜索引擎来用。大模型本身的知识是静态的，但结合RAG就可以使用搜索引擎的API，对要询问的问题做“阅读理解”自然要比人工手动地去搜寻相关的网页和整理要省力不少。
2. 对于分块和检索的部分，有点像当初计算机组成原理中学到的存储的机制。RAG的代码相比于微调，给人的感觉更多像调包，但要将其完成很好的优化和实现出色的效果则需要花费更多的时间，并且要结合prompt工程。
